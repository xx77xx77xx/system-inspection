# 🏥 医院系统巡检机器人 - 使用手册

本项目是一款面向医院信息科、运维团队设计的轻量化系统状态监控与告警工具。通过定时任务对 URL、数据库、主机、端口进行全方位巡检，并提供 Web 可视化看板及企业微信机器人实时告警。

---

## 🌟 核心特性

- **多维巡检**：支持 Web接口 (HTTP/HTTPS)、Oracle/MySQL 数据库、主机 Ping、TCP/Telnet 端口连通性。
- **自动调度**：
  - **实时巡检**：服务启动后每 **5 分钟** 自动执行一次全量检查。
  - **每日日报**：每天 **09:00** 准时推送巡检汇总报告。
- **便携化运行**：支持一键构建本地 Python 环境，不依赖系统安装的 Python，支持 Win7/Win10 等多平台移植。
- **企业微信告警**：
  - **防抖机制**：连续失败 3 次才触发告警，避免偶发抖动。
  - **日报汇总**：每天 09:00 自动发送巡检汇总。
  - **自逾喜报**：系统从异常恢复后自动发送一次恢复通知。
  - **超长自截**：针对多项目巡检，自动处理企微 2048 字节长度限制。
- **可视化看板**：实时流式日志查看、在线配置修改、手动触发巡检、企微通道一键测试。
- **安全加固**：后台管理接口受 `admin/890622` 账号密码保护。

## 🚀 快速开始

### 1. 环境构建 (首次运行必看)

本项目支持 **Windows 7/10/11 (32位及64位)** 系统。

1. 进入项目根目录。
2. 右键点击 `setup_env.ps1` -> 选择 **“使用 PowerShell 运行”**。
3. 脚本会自动下载对应架构的 Python 3.8 嵌入版，并自动修复 Win7 的 TLS 下载限制及 site-packages 路径。
4. 等待控制台显示 `[+] 构建完成` 即可。

### 2. 启动服务

双击运行 `start.bat`。

- 窗口将显示当前环境检测结果。
- Web 管理系统将在后台启动 (默认端口 `8000`)。
- 窗口最后会暂停，提示您手动执行一波初始巡检。

### 3. 进入管理后台

在浏览器访问：`http://localhost:8000`

- **用户名**：`admin`
- **密码**：`890622`

## ⚙️ 详细配置说明

### 企业微信告警配置

在 **“🚨 企业微信机器人配置”** 选项卡下：

1. **Webhook URL**：填入机器人 Webhook 地址（形如 `https://qyapi.weixin.qq.com/.../send?key=xxx`）。
2. **Secret Key**：如果 URL 里已经带了 key，此处可留空；也可以将 key 单独填在这里。
3. **连通性测试**：点击“发送测试消息”，系统会先发一条验证消息，随后执行全量巡检并推送即时快照到群内。

### 巡检节点管理

在 **“⚙️ 巡检节点配置”** 下，点击对应的“新增”按钮进行增删改：

- **Web接口**：监控主页、HIS/PACS 接口是否返回 200。
- **数据库**：配置连接字符串、账号密码。程序会尝试建立连接。
- **主机/端口**：Ping 核心交换机 IP，检查 Redis/中间件端口存活。

> [!IMPORTANT]
> **切记**：修改任何配置后，必须滚动到页面底部点击 **“💾 保存当前页面的系统级配置”** 才能生效。

## 🖥️ 服务器部署建议

1. **防火墙配置**：请在服务器防火墙中开放 `8000` 端口，以便其他终端访问。
2. **公网/跨网段访问**：在 `start.bat` 中已配置 `--host 0.0.0.0`，支持通过 IP 直接访问。
3. **设置为系统服务 (推荐)**：
   - 生产环境下，建议使用 [NSSM](https://nssm.cc/) 工具将 `start.bat` 注册为 Windows 服务。
   - 这样可以实现服务器开机自启，且后台静默运行，无需登录用户界面。
